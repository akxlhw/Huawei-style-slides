### 1. 金字塔原理（Pyramid Principle）

> **出处**：Barbara Minto，1970s，麦肯锡内部培训教材  
> **核心**：结论先行，以上统下，归类分组，逻辑递进  
> **一句话**：先给答案，再给理由

#### 在HR汇报中的应用

**所有给高管/管理层的汇报都应遵循金字塔原理**，尤其是：

| HR场景 | 金字塔应用方式 |
|--------|--------------|
| **高管人力简报** | 第1页放核心结论（"Q3招聘完成率60%，低于目标20个百分点，主因是技术岗渠道单一"），后面再放数据支撑 |
| **年度人力规划** | 第1页放总目标（"2025年净增150人，人力成本控制在营收12%以内"），再按部门拆解 |
| **离职分析报告** | 第1页放核心发现（"Q3离职率15%，超出行业均值5个点，根因是薪酬竞争力不足"），再展开分析 |
| **调薪方案** | 第1页放建议结论（"建议技术岗普调8%，关键岗位单独申请15%留人预算"），再放市场数据 |

#### 金字塔结构在PPT中的落地

```
第1页：封面
第2页：核心结论页（Executive Summary）
    ├─ 一句话结论
    ├─ 3个关键支撑点
    └─ 建议行动
第3页：支撑点1详述 + 数据
第4页：支撑点2详述 + 数据
第5页：支撑点3详述 + 数据
第6页：行动计划
```

**关键口诀**：如果高管只有30秒，看完第2页就够了。

### 2. MECE原则（Mutually Exclusive, Collectively Exhaustive）

> **出处**：麦肯锡问题解决方法论核心  
> **核心**：分类时做到**相互独立，完全穷尽**——不重叠，不遗漏  
> **一句话**：拆解问题时，每个维度只归一类，加起来是全部

#### 在HR汇报中的应用

**所有涉及"分类"的HR分析都必须MECE**，否则结论会被挑战：

| HR场景 | MECE应用示例 |
|--------|-------------|
| **离职原因分析** | 按MECE分为：薪酬因素 / 发展因素 / 管理因素 / 个人因素 / 外部因素（五类互斥且穷尽） |
| **招聘渠道分析** | 按MECE分为：自有渠道（官网/内推）/ 付费渠道（猎聘/BOSS）/ 社交渠道（脉脉/领英）/ 校园渠道 |
| **人力成本分析** | 按MECE分为：固定薪酬 / 变动薪酬 / 福利 / 培训 / 招聘 / 离职成本 |
| **岗位序列划分** | 按MECE分为：管理序列（M）/ 专业序列（P）/ 技术序列（T）/ 操作序列（O） |
| **培训需求分析** | 按MECE分为：新员工 / 高潜人才 / 管理者 / 转岗人员 / 全员通识 |

#### MECE常见错误（HR场景）

| 错误示例 | 问题 | 正确MECE分法 |
|---------|------|-------------|
| 离职原因分为：钱少、加班多、领导差、想回家 | "加班多"和"领导差"可能同时存在（不互斥） | 改为维度：薪酬竞争力 / 工作负荷 / 管理质量 / 职业发展 / 个人原因 |
| 招聘渠道分为：BOSS、猎聘、校招、其他 | "其他"是垃圾筐（不完全穷尽） | 明确列出所有渠道：BOSS/猎聘/智联/前程/校招/内推/猎头/社交/官网 |

### 3. 逻辑树（Issue Tree）

> **出处**：麦肯锡问题拆解核心工具  
> **核心**：把大问题拆成小问题，直到可执行；有两种树：问题树（Why）和方案树（How）  
> **一句话**：大问题摸不着，拆成小问题就有解了

#### 在HR汇报中的应用

| HR场景 | 逻辑树类型 | 拆解示例 |
|--------|-----------|---------|
| **招聘周期长** | 问题树（Why） | 招聘周期长 → 是简历少？还是面试慢？还是offer被拒？→ 逐层往下拆到根因 |
| **离职率高** | 问题树（Why） | 离职率高 → 是入职6个月内流失？还是3年以上老员工？→ 不同群体根因不同 |
| **搭建新团队** | 方案树（How） | 3个月内搭建10人AI团队 → 招聘6人+内部转岗3人+外包1人 → 每人具体到渠道和时间 |
| **降本增效** | 方案树（How） | 人力成本降10% → 优化招聘流程省20% + 培训内生化省30% + 调整外包比例省50% |

#### 在PPT中的呈现方式

```
问题：技术岗平均招聘周期60天，远超目标30天

        ┌─────────────────────────┐
        │  招聘周期60天 vs 目标30天  │
        └───────────┬─────────────┘
         ┌──────────┼──────────┐
         ▼          ▼          ▼
    ┌─────────┐ ┌─────────┐ ┌─────────┐
    │简历获取慢│ │面试流程长│ │Offer接受低│
    │  (20天)  │ │  (25天)  │ │  (15天)  │
    └────┬────┘ └────┬────┘ └────┬────┘
         │             │             │
    ┌────┴────┐   ┌────┴────┐   ┌────┴────┐
    ▼         ▼   ▼         ▼   ▼         ▼
 渠道少   JD不清 面试官忙  流程冗余 薪酬低  对手抢
```

### 4. SCQA框架（Situation-Complication-Question-Answer）

> **出处**：Barbara Minto，麦肯锡叙事标准框架  
> **核心**：情境→冲突→问题→答案，制造"认知缺口"再填上  
> **一句话**：先让听众觉得"确实是个问题"，再说"我有办法"

#### 在HR汇报中的应用

**所有"申请资源/申请变革"的HR汇报都适合SCQA**：

| HR场景 | SCQA应用 |
|--------|---------|
| **申请新增招聘渠道预算** | S：目前技术岗靠BOSS直聘和内推 → C：BOSS转化率从8%降到3%，内推源枯竭 → Q：如何保障Q4技术岗到位？ → A：新增猎聘+脉脉付费渠道，预算X万，预计解决50%缺口 |
| **申请调薪** | S：目前技术岗薪酬在市场50分位 → C：Q3技术岗离职率25%，离职访谈中70%提到薪酬 → Q：如何降低技术岗流失？ → A：申请技术岗调薪至市场75分位，预算X万，预计离职率降至15% |
| **申请上新HR系统** | S：目前用Excel管理招聘流程 → C：200+在招岗位，Excel频繁出错，协作效率低 → Q：如何提升招聘管理效率？ → A：上ATS系统，月费X，预计释放1个HR全职人力 |
| **组织架构调整提案** | S：目前技术部50人分5组汇报给3个总监 → C：跨组协作项目推进慢，总监间资源争抢 → Q：如何提升技术部协作效率？ → A：合并为3大中心+1个平台组，调整汇报线 |

#### SCQA在PPT中的页结构

```
第1页：封面
第2页：S - 情境（大家都认可的现状）
第3页：C - 冲突（打破预期的问题/变化）
第4页：Q - 核心问题（如果不解决会怎样）
第5页：A - 你的方案（方案概述）
第6页：A - 方案详述（How/预算/时间）
第7页：A - 预期效果（量化ROI）
第8页：下一步行动请求
```

### 5. 七步成诗（The Seven-Step Problem-Solving）

> **出处**：麦肯锡问题解决标准流程  
> **核心**：定义问题→拆解问题→优先级排序→工作计划→分析→综合→沟通  
> **一句话**：大问题不知道怎么下手？按这七步走

#### 在HR汇报中的应用

**所有"复杂项目复盘"和"年度规划"类汇报都可用七步成诗**：

| 步骤 | 含义 | HR应用示例（年度人力规划） |
|------|------|----------------------|
| **1. 定义问题** | 明确要回答什么问题 | "2025年如何配置人力资源，支撑业务增长30%的同时人力成本不超出营收12%？" |
| **2. 拆解问题** | 用逻辑树/MECE拆分 | 拆为：招聘规划 / 调配优化 / 成本控制 / 效率提升 四大模块 |
| **3. 优先级排序** | 抓重点，28原则 | 技术岗招聘（影响营收）> 管理层继任（影响稳定）> 通用培训（可延后） |
| **4. 工作计划** | 分工和时间表 | HRBP负责业务线需求、COE负责政策、SSC负责数据 |
| **5. 分析** | 收集数据，深入分析 | 各部门人效数据、市场薪酬数据、离职根因访谈 |
| **6. 综合** | 提炼洞察，形成结论 | "技术岗需净增50人，管理岗需内部提拔3人，整体成本可控" |
| **7. 沟通** | 汇报并推动落地 | 按金字塔结构汇报，第1页放结论 |

#### 七步成诗在PPT中的呈现

```
第1页：封面
第2页：我们在解决什么问题（Step 1）
第3页：问题拆解框架（Step 2，逻辑树图）
第4页：优先级矩阵（Step 3，影响vs紧急程度）
第5页：分析过程与关键发现（Step 5）
第6页：核心结论与建议（Step 6）
第7页：行动计划（Step 7，谁/做什么/何时）
```

### 6. 九宫格矩阵（9-Box Grid）

> **出处**：麦肯锡/通用电气联合开发，后成为人才管理标准工具  
> **核心**：绩效（过去）× 潜力（未来）两个维度，各分高中低三档，形成9格  
> **一句话**：把人才分九类，不同类给不同策略

#### 九宫格定义

```
            低绩效        中绩效        高绩效
         ┌──────────┬──────────┬──────────┐
    高潜力 │  待提升者  │   emerging  │   明星    │
         │(6-12个月)│  star      │  (重点保留)│
         ├──────────┼──────────┼──────────┤
    中潜力 │  问题员工  │  稳定贡献者 │   核心骨干  │
         │(需决策)   │  (维持)     │  (激励)    │
         ├──────────┼──────────┼──────────┤
    低潜力 │  淘汰对象  │  熟练员工   │   老黄牛   │
         │(尽快处理)  │  (维持)     │  (认可但   │
         │           │            │   不提拔)  │
         └──────────┴──────────┴──────────┘
```

#### 在HR汇报中的应用

| HR场景 | 九宫格应用 |
|--------|-----------|
| **年度人才盘点** | 全员入格，形成人才地图，汇报给高管 |
| **继任规划** | 高绩效+高潜力 = 重点培养接班人；高绩效+中潜力 = 专业线深耕 |
| **培训资源分配** | 高潜力员工优先分配高成本培训（如EMBA、海外轮岗） |
| **裁员决策** | 低绩效+低潜力优先列入优化名单 |
| **关键人才保留** | 高绩效+高潜力 = 单独制定保留方案（薪酬/发展/特殊项目） |

#### 九宫格在PPT中的呈现

```
第1页：封面
第2页：盘点方法论（九宫格模型说明）
第3页：人才分布九宫格（气泡图，气泡大小=人数）
第4页：高潜明星名单（姓名/岗位/发展计划）
第5页：核心骨干保留方案（薪酬/发展/激励）
第6页：问题员工决策清单（改善计划或淘汰时间表）
第7页：人才缺口与补充计划
第8页：培养与发展预算
```

### 7. 7S模型（McKinsey 7S Framework）

> **出处**：Richard Pascale & Anthony Athos，1980s，麦肯锡推广  
> **核心**：7个相互关联的组织要素——战略、结构、制度、风格、人员、技能、共同价值观  
> **一句话**：改组织不能只改架构，7个要素要一起动

#### 7S要素详解

| 要素 | 英文 | 含义 | HR关注点 |
|------|------|------|---------|
| **战略** | Strategy | 组织的目标和方向 | 人力战略是否对齐业务战略 |
| **结构** | Structure | 组织架构和汇报线 | 架构是否支撑战略落地 |
| **制度** | Systems | 流程和制度 | 绩效/薪酬/晋升制度是否配套 |
| **共同价值观** | Shared Values | 企业文化 | 价值观是否支撑变革 |
| **风格** | Style | 领导风格 | 管理层的领导方式是否一致 |
| **人员** | Staff | 人才配置 | 关键岗位是否有合适的人 |
| **技能** | Skills | 组织能力 | 团队的能力短板在哪里 |

#### 在HR汇报中的应用

| HR场景 | 7S应用 |
|--------|--------|
| **组织诊断** | 用7S全面扫描组织健康度，找出短板要素 |
| **并购后整合** | 用7S评估被并购公司，制定整合方案 |
| **架构调整提案** | 不只说改架构，还要说制度/流程/文化怎么配套改 |
| **文化变革** | 用7S说明文化不是单独改价值观，还要改制度/流程/领导行为 |
| **新建业务单元** | 用7S确保新业务从战略到人才的全要素配套 |

#### 7S在PPT中的呈现

```
第1页：封面
第2页：7S诊断框架说明
第3页：现状评估雷达图（7个维度的评分）
第4页：核心问题诊断（哪几个S是短板）
第5页：整改方案（7S联动的具体措施）
第6页：实施路线图
第7页：预期效果
```

### 8. 3C模型（Company, Customer, Competitor）

> **出处**：大前研一/麦肯锡战略分析工具  
> **核心**：从公司自身、客户、竞争者三个维度分析战略定位  
> **一句话**：知己知彼，还要看自己

#### 在HR汇报中的应用（适配为人才竞争版3C）

| 原3C | HR适配版 | 含义 | 分析内容 |
|------|---------|------|---------|
| **Company** | **Company（公司）** | 自身人才现状 | 人才结构、能力地图、人效数据、雇主品牌 |
| **Customer** | **Candidate（候选人）** | 目标人才 | 目标人群画像、求职偏好、渠道触点 |
| **Competitor** | **Competitor（竞争对手）** | 人才竞争者 | 同行薪酬水平、雇主品牌、人才策略 |

#### 具体HR场景

| HR场景 | 3C应用 |
|--------|--------|
| **薪酬竞争力分析** | C1：我司薪酬结构 / C2：目标人才的薪酬期望 / C3：竞品公司的薪酬水平 |
| **招聘策略制定** | C1：我司雇主品牌优势 / C2：目标候选人的求职渠道 / C3：竞品公司的招聘打法 |
| **人才保留方案** | C1：我司能提供的（薪酬/发展/文化） / C2：员工想要什么 / C3：竞品在挖人的手段 |
| **雇主品牌建设** | C1：我司独特价值主张 / C2：目标人才在意什么 / C3：竞品雇主品牌形象 |

### 9. 组织健康指数 OHI（Organizational Health Index）

> **出处**：麦肯锡专有工具，对600+公司、700万+员工的研究  
> **核心**：用9大维度、37个子项量化评估组织健康度，健康公司股东回报是不健康公司的3倍  
> **一句话**：组织健康不是玄学，是可以量化的

#### OHI 9大维度

| 维度 | 含义 | HR汇报中可以量化的指标 |
|------|------|---------------------|
| **方向** | 战略清晰且被理解 | 员工对战略的理解度调研 |
| **领导力** | 管理层授权和激励 | 360评估得分、管理span |
| **文化** | 文化驱动行为 | 价值观践行度调研 |
| **创新** | 鼓励新想法 | 内部创新提案数、采纳率 |
| **问责** | 目标明确且可追溯 | OKR完成率、绩效区分度 |
| **协调** | 跨部门协作顺畅 | 跨部门项目满意度 |
| **能力** | 关键能力到位 | 关键岗位胜任率 |
| **动力** | 员工有动力 | 敬业度得分、eNPS |
| **外部导向** | 关注客户和市场 | 客户导向行为调研 |

#### 在HR汇报中的应用

| HR场景 | OHI应用 |
|--------|--------|
| **年度组织诊断报告** | 用OHI 9维度全面评估，对标行业 |
| **敬业度调研汇报** | 用OHI框架解读数据，不只是列分数 |
| **组织变革方案** | 变革前后OHI对比，证明变革效果 |
| **HR战略规划** | 用OHI识别组织短板，制定HR priorities |
| **向董事会汇报HR价值** | "OHI每提升1分，营收增长X%" |

### 10. 人才-价值挂钩（Talent-to-Value Approach）

> **出处**：麦肯锡人才管理方法论  
> **核心**：人才投入要跟业务价值挂钩——关键岗位的人才是ROI最高的投资  
> **一句话**：不是所有岗位都值得重金投入，要把钱花在创造价值的地方

#### 核心逻辑

```
业务价值链分析 → 识别关键岗位（Value-Critical Roles）
    ↓
评估在岗人员匹配度
    ↓
不匹配 → 替换/培养
匹配 → 重点保留/激励
    ↓
持续跟踪岗位价值贡献
```

#### 在HR汇报中的应用

| HR场景 | 人才-价值挂钩应用 |
|--------|-----------------|
| **关键人才保留方案** | 先识别哪些岗位是"价值关键"，再给这些岗位上的人单独制定保留方案 |
| **招聘优先级排序** | 不是哪个岗位最紧急就招哪个，而是哪个岗位对业务价值最大优先招 |
| **培训预算分配** | 高价值岗位的人优先分配高成本培训 |
| **高管薪酬方案** | CEO/CFO等关键岗位薪酬与业务价值指标直接挂钩 |
| **人才投资ROI汇报** | "去年在X岗位投入培训费用Y万，该岗位产出提升Z%，ROI为A" |

### 6.1 招聘漏斗分析（AIDA模型精细化应用）

> **理论来源**：AIDA模型（Attention→Interest→Desire→Action）  
> **HR适配**：招聘漏斗 = 曝光→投递→筛选通过→面试通过→offer接受→入职留存

#### 标准招聘漏斗（6层MECE分解）

```
┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐    ┌─────────┐
│  渠道    │ → │  简历    │ → │  初筛    │ → │  面试    │ → │  Offer  │ → │  入职    │
│  曝光    │    │  投递    │    │  通过    │    │  通过    │    │  接受    │    │  留存    │
│  100%   │    │   15%   │    │   40%   │    │   30%   │    │   80%   │    │   90%   │
└─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘    └─────────┘
    │                │                │                │                │                │
    ▼                ▼                ▼                ▼                ▼                ▼
 渠道ROI分析      JD吸引力        筛选标准        面试官能力      薪酬竞争力      入职体验
```

#### 关键转化率计算公式

| 指标 | 公式 | 健康基准 |
|------|------|---------|
| **简历转化率** | 投递数 ÷ 曝光量 × 100% | ≥10% |
| **初筛通过率** | 初筛通过数 ÷ 投递数 × 100% | 30%-50% |
| **面试通过率** | 面试通过数 ÷ 初筛通过数 × 100% | 25%-40% |
| **Offer接受率** | 接受数 ÷ 发出数 × 100% | ≥75% |
| **入职到岗率** | 实际入职 ÷ 接受offer数 × 100% | ≥90% |
| **整体漏斗效率** | 最终入职 ÷ 总投递 × 100% | ≥1.5% |
| **单人均招聘成本** | 总招聘成本 ÷ 入职人数 | 行业均值：年薪的15-25% |

#### 漏斗异常诊断（逻辑树应用）

```
某岗位招聘周期60天（目标30天）
         │
    ┌────┴────┐
    ▼         ▼
简历量少?   转化率低?
    │         │
    ▼         ▼
┌───────┐  ┌────────┐
│渠道问题│  │JD问题  │
│预算不足│  │要求过高│
│市场人才│  │薪酬无竞争力│
│稀缺    │  │        │
└───────┘  └────────┘
```

#### PPT汇报模板

```
第1页：封面（XX岗位招聘漏斗分析报告）
第2页：核心结论（金字塔）
    "技术岗Java工程师招聘周期60天，超目标100%，
     瓶颈在简历获取环节（转化率仅3%，目标10%）"

第3页：招聘漏斗全图（6层漏斗可视化）
    每层标注：数量、转化率、与目标对比（红/黄/绿）

第4页：瓶颈环节详析（MECE根因）
    简历少 → 根因拆解：
    ├─ 渠道覆盖不足（目前仅BOSS，缺猎聘/脉脉）
    ├─ JD关键词不匹配（未覆盖"Spring Cloud"等热词）
    └─ 雇主品牌弱（公司主页评分3.2，竞品4.5）

第5页：改进方案与预期效果
    ├─ 新增猎聘渠道 → 预计简历+40%
    ├─ 优化JD关键词 → 预计搜索曝光+60%
    └─ 预算：X万 / 预期周期：60天→35天

第6页：行动计划（甘特图）
    第1周：开通猎聘账号
    第2周：完成JD优化
    第3周：启动内推激励
    第4周：数据复盘
```

### 6.2 招聘渠道ROI分析（MECE + 3C模型）

> **理论来源**：MECE原则（渠道分类不重叠不遗漏）+ 3C模型（对标竞品渠道策略）  
> **核心问题**：哪些渠道值得继续投？哪些该砍掉？预算怎么重新分配？

#### 渠道MECE分类（确保全覆盖）

```
招聘渠道（MECE分解）
│
├─ 自有渠道（免费/低成本）
│   ├─ 官网招聘页
│   ├─ 内部推荐
│   └─ 人才库激活
│
├─ 付费垂直平台
│   ├─ BOSS直聘
│   ├─ 猎聘
│   ├─ 智联招聘
│   └─ 前程无忧
│
├─ 社交平台
│   ├─ 脉脉
│   ├─ LinkedIn
│   └─ 微信群/公众号
│
├─ 猎头渠道
│   ├─ 高端猎头（年薪30万+）
│   └─ RPO批量招聘
│
└─ 校园渠道
    ├─ 春招/秋招
    └─ 实习生转正
```

#### 渠道ROI计算公式

| 指标 | 公式 | 用途 |
|------|------|------|
| **单简历成本** | 渠道费用 ÷ 收到简历数 | 评估引流效率 |
| **单面试成本** | 渠道费用 ÷ 面试人数 | 评估简历质量 |
| **单入职成本** | 渠道费用 ÷ 入职人数 | 评估最终ROI |
| **渠道贡献率** | 该渠道入职数 ÷ 总入职数 × 100% | 评估渠道占比 |
| **渠道性价比评分** | 贡献率 ÷ 成本占比 | >1说明性价比高 |

#### 渠道ROI对比表（PPT可直接复制）

| 渠道 | 费用(元) | 简历数 | 初筛通过 | 面试 | Offer | 入职 | 单入职成本 | 贡献率 | 性价比 |
|------|---------|--------|---------|------|-------|------|-----------|--------|--------|
| BOSS直聘 | 20,000 | 200 | 80 | 32 | 12 | 10 | 2,000 | 40% | 1.6 |
| 猎聘 | 30,000 | 80 | 40 | 20 | 8 | 6 | 5,000 | 24% | 0.8 |
| 内推 | 5,000 | 30 | 24 | 15 | 10 | 9 | 556 | 36% | 3.6 |
| 智联 | 10,000 | 120 | 24 | 8 | 2 | 0 | ∞ | 0% | 0 |

**结论**：内推性价比最高（3.6），应加大激励；智联零产出，建议暂停；猎聘成本高但质量尚可，维持观察。

#### 渠道优化汇报PPT结构

```
第1页：封面（Q3招聘渠道ROI分析报告）
第2页：核心结论（金字塔）
    "Q3招聘总投入6.5万，内推渠道ROI最高（3.6分），
     智联零产出建议停投，释放预算加投BOSS和内推"

第3页：渠道MECE分类全景图
第4页：各渠道ROI对比表（上表）
第5页：3C对标（竞品渠道策略）
    Competitor A：内推占比50%（激励政策XX）
    Competitor B：校招为主（提前锁定人才）
    我司：内推仅占20% → 差距 = 机会

第6页：优化方案
    ├─ 停投智联（节省1万/季度）
    ├─ 增加BOSS预算50%（+1万）
    ├─ 内推激励翻倍（从500/人→1000/人，预算+0.5万）
    └─ 净效果：成本不变，预计入职数+30%

第7页：下季度渠道预算分配
```

### 6.3 人员调配决策框架（逻辑树 + 九宫格）

> **理论来源**：逻辑树（该不该调？怎么调？）+ 九宫格（调谁？）  
> **核心问题**：什么情况下该内部调配？什么情况下该外部招聘？调谁？怎么沟通？

#### 内部调配 vs 外部招聘 决策树

```

    │
    ▼

│ 现有团队有人可胜任？ │

    ▼              ▼

   ▼      \        ▼       \  ▼

              │
              ▼
        ┌──────────┐

        └────┬─────┘
           是 /  \否
            ▼    \ ▼
        外部招聘   执行调配
```

#### 调配决策的MECE评估维度

| 维度 | 评估内容 | 权重 |
|------|---------|------|
| **能力匹配度** | 现岗能力评分 vs 目标岗要求 | 30% |
| **绩效表现** | 连续2个季度绩效≥B | 25% |
| **发展意愿** | 员工本人意愿（1-5分） | 20% |
| **磨合成本** | 新岗上手预计时间 | 15% |
| **替代成本** | 原岗位替代难度 | 10% |

**决策规则**：总分≥4.0分且员工意愿≥3分 → 推荐调配；总分<3.0分 → 外部招聘。

### 6.4 招聘需求优先级排序（2×2矩阵）

> **理论来源**：2×2优先级矩阵（影响 × 紧急度）+ 人才-价值挂钩  
> **核心问题**：需求永远做不完，先做哪个？

#### 招聘需求优先级矩阵

```
            低紧急           高紧急
              │                │
    高影响    │  战略储备      │  立即行动（第一优先）│
              │  （提前布局）   │  如：CEO助理、核心架构师│
              │                │                    │
    ──────────┼────────────────┼────────────────────┤
              │                │                    │
    低影响    │  暂缓/冻结      │  快速解决（第二优先）│
              │  （不做）       │  如：实习生、辅助岗   │
              │                │                    │
```

**四象限处理策略：**

| 象限 | 处理策略 | 审批权限 |
|------|---------|---------|
| **高影响+高紧急** | 立即启动，优先分配资源 | 总经理直批 |
| **高影响+低紧急** | 纳入计划，提前布局 | HRD审批 |
| **低影响+高紧急** | 简化流程，快速解决 | 部门经理审批 |
| **低影响+低紧急** | 冻结或取消 | 统一冻结 |

#### 影响度评估维度（MECE）

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| 对业务营收的影响 | 30% | 直接创收岗5分 / 支撑岗3分 / 后勤岗1分 |
| 项目交付依赖度 | 25% | 关键路径5分 / 非关键3分 / 无影响1分 |
| 现有团队负荷度 | 20% | 超负荷5分 / 饱和3分 / 有富余1分 |
| 市场人才稀缺度 | 15% | 极度稀缺5分 / 一般3分 / 充足1分 |
| 替代成本 | 10% | 极高5分 / 中等3分 / 低1分 |

#### 紧急度评估标准

| 等级 | 定义 | 举例 |
|------|------|------|
| 5分 | 1周内必须到位 | 关键项目核心成员离职 |
| 3分 | 1个月内到位 | 新业务启动团队组建 |
| 1分 | 3个月内到位 | 常规替补/团队扩建 |

#### 优先级排序PPT结构

```
第1页：封面（Q4招聘需求优先级排序）
第2页：总需求一览
    共收到28个招聘需求 → 经评估：
    立即行动(4) / 纳入计划(12) / 快速解决(8) / 建议冻结(4)

第3页：评估方法论（2×2矩阵 + MECE维度）
第4页：立即行动清单（TOP 4）
    岗位 / 影响度 / 紧急度 / 理由

第5页：建议冻结清单及理由
    岗位 / 冻结理由 / 解冻条件

第6页：资源分配方案
    招聘团队3人 × 优先级分配时间

第7页：时间表（甘特图）
```

### 6.5 调配沟通方案（SCQA + 金字塔）

> **理论来源**：SCQA框架 + 金字塔原理  
> **核心问题**：怎么跟被调配的员工谈？怎么跟原部门经理解释？

#### 员工沟通话术模板（SCQA结构）

```
【开场 - Situation】
"小明，首先肯定你过去一年的工作，
 连续两个季度绩效A，招聘完成率团队第一。"

【转折 - Complication】
 "有个情况和你沟通：公司业务部正在快速扩张，
  目前3个城市缺HRBP支持，管理层一致认为你是最佳人选。"

【问题 - Question】
 "这个机会对你意味着：直接接触业务核心、
  职业发展空间更大，但也会更有挑战。
  我想听听你的想法？"

【回答 - Answer】
 「如果同意」→ "太好了！接下来我们聊一下 transition 计划..."
 「如果犹豫」→ "理解你的顾虑。我们安排你和业务部张总聊一下，
              了解一下具体工作内容，再做决定，好吗？"
 「如果拒绝」→ "完全尊重你的选择。能告诉我主要的顾虑是什么吗？
              看看我们能不能一起解决。"
```

#### 原部门经理沟通话术（金字塔结构）

```
【结论先行】
"李经理，有个事提前和你通气：
 小明预计2周后调到业务部做HRBP。
 我理解这会对你团队有影响，所以来找你商量过渡方案。"

【理由支撑】
"原因有两个：
 1. 业务部扩张是Q4公司战略重点，需要最得力的人支持
 2. 小明的职业发展也需要这个机会，公司层面想重点培养他

【解决方案】
"针对你团队的缺口，我们准备这样安排：
 1. 招聘部正在物色接替人选，预计1周内到岗
 2. 小明会留2周做交接，确保平稳过渡
 3. 过渡期间如有紧急事务，你可以直接找我对接"

【征求意见】
"你看这样安排OK吗？还有什么需要我们配合的？"
```

#### 调配沟通PPT（如需正式汇报审批）

```
第1页：封面
第2页：调配背景与决策逻辑（金字塔：结论先行）
第3页：人选评估依据（九宫格位置 + 匹配度评分）
第4页：员工沟通记录摘要
第5页：原部门过渡方案
第6页：新岗位培养计划（30/60/90天）
第7页：风险与应对
```

### 6.6 关键岗位紧急招聘方案（七步成诗加速版）

> **理论来源**：七步成诗（压缩到48小时内完成）  
> **核心场景**：关键岗位突然离职 / 新业务紧急启动，必须在X天内招到人

#### 48小时紧急招聘作战计划

| 时间 | 动作 | 负责人 | 麦肯锡工具 |
|------|------|--------|-----------|
| **0-4h** | 需求确认：岗位JD、必须条件、可妥协项 | 用人部门+HRBP | MECE（条件分级） |
| **4-8h** | 渠道全开：ATS发布+猎头同步+内推广播+社交扩散 | 招聘专员 | 3C（对标竞品速度） |
| **8-24h** | 简历筛选+初面安排（可电话初筛） | 招聘专员+用人部门 | 漏斗（加速转化） |
| **24-36h** | 终面+决策（集中面试日） | 用人部门+HRD | 九宫格（快速评估） |
| **36-48h** | Offer谈判+发出 | HRBP | SCQA（说服候选人） |

#### 紧急招聘PPT（事后复盘用）

```

    C：该岗位支撑Q4核心项目，空缺将导致项目延期2周
    Q：如何在48小时内找到接替者？

第3页：作战计划（甘特图，精确到小时）
第4页：执行结果
    收到简历：XX份 → 初筛：XX人 → 面试：XX人 → Offer：X人 → 入职：X人
    总用时：43小时（目标48小时）✓

第5页：关键成功因素
    1. 内推渠道快速激活（XX小时内收到推荐）
    2. 用人部门配合度高（当天安排面试）
    3. 薪酬预审批前置（无需层层审批）

第6页：可复用的机制建议
    1. 关键岗位必须建立AB角
    2. 建立"紧急招聘绿色通道"审批流程
    3. 维护"人才储备库"，定期激活
```

### 6.7 招聘质量分析（试用期留存率 × 绩效关联）

> **理论来源**：逻辑树（招聘质量差的表现→根因）+ MECE  
> **核心问题**：招进来的人不好用，是哪里出了问题？

#### 招聘质量评估指标体系

| 阶段 | 指标 | 公式 | 健康基准 |
|------|------|------|---------|
| **试用期** | 试用期通过率 | 转正人数 ÷ 入职人数 × 100% | ≥85% |
| **短期** | 6个月留存率 | 满6个月人数 ÷ 入职人数 × 100% | ≥80% |
| **中期** | 1年留存率 | 满1年人数 ÷ 入职人数 × 100% | ≥70% |
| **绩效** | 高绩效占比 | 绩效A/B人数 ÷ 入职人数 × 100% | ≥60% |
| **成本** | 单次成功招聘成本 | 总成本 ÷ 最终转正人数 | 低于年薪20% |

#### 招聘质量问题诊断树

```
试用期留存率低（<80%）
    │
    ├─ 入职1个月内离职 → 期望落差（JD/面试过度承诺）
    │
    ├─ 入职2-3个月离职 → 融入失败（导师制缺失/团队排斥）
    │
    ├─ 试用期不通过 → 能力不匹配（面试评估不准）
    │
    └─ 转正后3个月内离职 → 发展落差（晋升通道不清）
```

#### 招聘质量PPT结构

```
第1页：封面（2024年招聘质量分析报告）
第2页：核心结论
    "全年入职80人，试用期通过率78%（目标85%），
     根因：面试评估标准不统一 + 部分JD过度美化"

第3页：招聘漏斗全链路质量指标
    曝光→投递→面试→offer→入职→转正→1年留存
    每层标注转化率

第4页：问题诊断（逻辑树）
    试用期12人未通过 → 根因拆解

第5页：改进方案
    1. 结构化面试评分卡（统一标准）
    2. JD真实性审核（HRBP复核）
    3. 入职导师制（每人配buddy）

第6页：预期效果与时间表
```

### 6.8 招聘调配岗位常用数据看板（MECE指标全集）

> 以下指标按MECE原则覆盖招聘调配全链路，可作为周报/月报的固定指标库。

#### 招聘模块指标体系

```
招聘效率（Time-based）
├─ 平均招聘周期（收到需求到入职天数）
├─ 各岗位招聘周期（按岗位序列拆分）
├─ Offer发放到接受平均天数
└─ 需求提出到启动招聘平均天数

招聘效果（Result-based）
├─ 招聘完成率（实际入职 ÷ 需求数）
├─ 招聘满足率（到岗人数 ÷ 计划数）
├─ 试用期通过率
├─ 6个月留存率
└─ 招聘渠道贡献分布

招聘成本（Cost-based）
├─ 总招聘费用
├─ 单人均招聘成本
├─ 单渠道ROI
├─ 猎头费用占比
└─ 招聘费用占人力成本比

招聘质量（Quality-based）
├─ 新员工绩效分布
├─ 新员工晋升比例（1年内）
├─ 用人部门满意度评分
└─ 新员工360度反馈
```

#### 调配模块指标体系

```
调配效率
├─ 调配响应周期（需求提出到人员到位天数）
├─ 内部调配占比（内部调配 ÷ 总需求数）
└─ 调配审批平均周期

调配效果
├─ 调配人员试用期通过率
├─ 调配人员1年留存率
├─ 调配后绩效变化（前后对比）
└─ 调配满意度（调出部门/调入部门/本人）

调配成本
├─ 内部调配成本
├─ 外部招聘成本
└─ 调配节省率（内部成本 ÷ 外部成本）
```
